# 中国方便面世界杯｜Vibe Coding 策划方案

## 1. 项目概述

**项目名称：** 中国方便面世界杯  
**宣传语：** 32 款方便面，只能留下一碗。  
**产品形式：** 移动端优先的网页小游戏  
**核心玩法：** 用户从 32 款中国市场常见方便面中，通过两两对决、单败淘汰，最终选出自己的方便面冠军。

项目只设置一种完整的“世界杯模式”：

- 32 款方便面参赛
- 共进行 31 场选择
- 依次经历 32 强、16 强、8 强、半决赛和决赛
- 最终生成冠军结果和完整淘汰赛长图

---

## 2. 产品目标

1. 让用户在 3～5 分钟内完成一届方便面世界杯。
2. 每次操作只需在两款方便面中选择一款。
3. 通过经典产品、怀旧元素和纠结对决提升趣味性。
4. 最终结果适合生成图片并分享到社交平台。
5. 首版不要求登录，不依赖后端。

---

## 3. 用户流程

```text
首页
  ↓
查看并确认 32 款参赛方便面
  ↓
生成随机对阵
  ↓
32 强：16 场
  ↓
16 强：8 场
  ↓
8 强：4 场
  ↓
半决赛：2 场
  ↓
决赛：1 场
  ↓
冠军结果页
  ↓
生成完整对阵长图 / 分享海报
```

用户总共需要做出 31 次选择。

## 4. 核心玩法

### 4.1 参赛阵容

每届比赛固定抽取 32 款方便面。

每款产品包含：

- 品牌
- 产品名称
- 包装图片
- 产品类型
- 口味标签
- 分类标签

建议建立一个 48～64 款的候选池，每次开局随机抽取 32 款。

候选品牌可包含：

- 康师傅
- 统一
- 白象
- 今麦郎
- 华丰
- 南街村
- 幸运
- 公仔面
- 合味道
- 出前一丁
- 五谷道场
- 汤达人
- 茄皇
- 小浣熊
- 魔法士
- 北京方便面

比较对象必须是具体产品，而不是品牌，例如：

- 康师傅红烧牛肉面
- 统一老坛酸菜牛肉面
- 白象老母鸡汤面
- 汤达人日式豚骨拉面

开发阶段可先使用文字占位图，正式上线前再补充经过核对的产品包装图。

---

### 4.2 赛前确认

比赛开始前展示本届 32 款参赛产品。

用户可以：

- 查看全部参赛产品
- 将没吃过的产品替换为候选池中的其他产品
- 重新随机生成整套参赛阵容
- 确认参赛名单并开始比赛

为避免反复刷新，单局最多允许替换 8 款产品。

---

### 4.3 对阵生成

首轮共生成 16 组对决。

对阵规则：

1. 32 款产品随机排列。
2. 尽量避免同品牌产品在第一轮相遇。
3. 热门产品尽量分散在不同半区。
4. 相同口味类别尽量不要集中在同一区域。
5. 每次重新开始都生成不同对阵。

不需要复杂的体育种子系统。随机排列后检测首轮冲突，并交换冲突选手即可。

---

### 4.4 比赛页面

每场比赛只展示两款方便面。

页面内容：

- 当前轮次，例如“32 强”
- 当前场次，例如“第 6 / 16 场”
- 整体进度条
- 左右两张方便面卡片
- 中间的“VS”
- 提示语：“选出你更愿意吃的一款”
- 撤销上一场按钮

卡片内容：

- 包装图片
- 品牌名称
- 产品名称
- 2～3 个口味标签

用户点击卡片后：

1. 被选中的卡片放大并高亮。
2. 显示“晋级”标签。
3. 失败卡片变暗。
4. 约 500 毫秒后自动进入下一场。
5. 胜者自动进入下一轮对应位置。
6. 比赛状态保存到 `localStorage`。

---

### 4.5 撤销功能

用户可以撤销上一场选择。

实现方式：

- 每次选择前保存一份比赛状态快照。
- 点击撤销后恢复最近一份快照。
- 撤销后重新显示上一场对决。
- 冠军产生后仍允许撤销决赛结果。

只需支持逐步撤销，不需要支持任意跳转修改历史结果。

## 5. 比赛轮次

| 轮次 | 参赛数量 | 比赛场数 |
|---|---:|---:|
| 32 强 | 32 | 16 |
| 16 强 | 16 | 8 |
| 8 强 | 8 | 4 |
| 半决赛 | 4 | 2 |
| 决赛 | 2 | 1 |
| 合计 | — | 31 |

下一轮位置规则：

- 当前比赛编号为偶数时，胜者进入下一场左侧。
- 当前比赛编号为奇数时，胜者进入下一场右侧。
- 下一场编号为 `Math.floor(currentMatchIndex / 2)`。

## 6. 结果页面

### 6.1 冠军展示

结果页中央展示：

- 冠军包装图
- 冠军产品名称
- 品牌名称
- 皇冠或奖杯元素
- “中国方便面世界杯冠军”
- 冠军击败过的全部对手

示例：

```text
中国方便面世界杯冠军

康师傅红烧牛肉面

它先后击败了：
华丰三鲜伊面
合味道海鲜杯面
白象老母鸡汤面
汤达人日式豚骨拉面
统一老坛酸菜牛肉面
```

---

### 6.2 方便面人格

根据冠军类型和用户选择记录，生成固定规则文案。

示例人格：

- **经典怀旧派**：偏爱传统牛肉面、三鲜面和老牌产品。
- **浓汤主义者**：多次选择豚骨汤、鸡汤和浓汤产品。
- **重口挑战者**：偏爱麻辣、酸辣和香辣口味。
- **干拌面信徒**：多次淘汰汤面，偏爱拌面和炒面。
- **海鲜汤底派**：偏爱海鲜、虾仁、鱼板等口味。
- **随机探索家**：晋级产品类别分散，没有明显单一偏好。

首版使用标签统计生成，不接入 AI。

---

### 6.3 完整对阵图

结果页面提供：

- 冠军晋级路线
- 完整 32 强淘汰赛树
- 生成纵向长图
- 重新开始比赛

手机端默认只展示冠军晋级路线，完整比赛树通过横向滑动或长图查看。

长图建议采用 9:16 或更长的纵向比例，参考歌曲世界杯的左右半区布局：

- 左半区从左向中间晋级
- 右半区从右向中间晋级
- 冠军位于页面中央
- 页面底部放置网站名称和二维码

## 9. 视觉设计

### 9.1 风格方向

整体视觉不直接复制歌曲世界杯的紫黑科技风，改为：

- 方便面包装感
- 复古小卖部氛围
- 热气、汤汁和面条元素
- 年轻、有趣、有比赛感
- 页面简洁，避免堆砌信息

### 9.2 推荐配色

- 深红：比赛氛围、辣味
- 暖黄：面条、冠军和高亮
- 米白：背景和食品感
- 深棕：文字、酱汁和边框
- 少量绿色：葱花和清汤标签

### 9.3 卡片设计

- 大圆角
- 明显包装图
- 轻微阴影
- 点击时缩放
- 晋级时出现黄色描边
- 失败时降低透明度
- 决赛卡片可以增加皇冠和金色光效

```ts
export type NoodleType =
  | "soup"

  | "snack";

export interface Noodle {
  id: string;

  image: string;
```

## 12. 核心比赛逻辑

```ts
function selectWinner(matchId: string, winnerId: string) {
  // 1. 保存当前状态快照
  // 2. 找到当前比赛
  // 3. 写入 winnerId
  // 4. 计算下一轮对应比赛和位置
  // 5. 将胜者填入下一轮
  // 6. 找到下一场尚未完成的比赛
  // 7. 若不存在下一场，则生成冠军
  // 8. 保存到 localStorage
}
```

下一轮位置：

```ts
const nextMatchIndex = Math.floor(currentMatch.matchIndex / 2);
const isLeftSlot = currentMatch.matchIndex % 2 === 0;

if (isLeftSlot) {
  nextMatch.leftNoodleId = winnerId;
} else {
  nextMatch.rightNoodleId = winnerId;
}
```

## 15. 验收标准

1. 用户可以在手机端完成完整的 32 强比赛。
2. 全程共进行 31 次选择。
3. 每场胜者能正确进入下一轮。
4. 刷新页面后能够恢复比赛进度。
5. 撤销操作不会破坏后续对阵。
6. 决赛胜者能正确成为冠军。
7. 结果页能展示冠军击败过的所有对手。
8. 能生成清晰的冠军分享海报。
9. 能生成完整的淘汰赛长图。
10. 常见手机宽度下不出现异常横向滚动。
11. 所有产品数据可以通过 `noodles.json` 增删。
12. 项目可以通过 TypeScript 检查并正常构建。
